## 1. SQL 숫자 처리 함수

SQL에서는 숫자를 반올림, 내림, 올림, 자르기 등으로 처리할 수 있는 함수들이 있다.

---

## ROUND()

`ROUND()`는 숫자를 반올림할 때 사용한다.

```sql
ROUND(값, 자릿수)
```

예시:

```sql
SELECT ROUND(123.4567, 2);
```

결과:

```text
123.46
```

소수 둘째 자리까지 남기고 반올림한다.

```sql
SELECT ROUND(123.4567, 0);
```

결과:

```text
123
```

정수 자리까지 반올림한다.

```sql
SELECT ROUND(123.4567, -1);
```

결과:

```text
120
```

십의 자리 기준으로 반올림한다.

---

## FLOOR()

`FLOOR()`는 값을 내림한다.

```sql
SELECT FLOOR(123.999);
```

결과:

```text
123
```

주의할 점은 음수에서도 "아래 방향"으로 내려간다는 것이다.

```sql
SELECT FLOOR(-123.1);
```

결과:

```text
-124
```

즉, 단순히 소수점만 제거하는 것이 아니라 해당 값보다 작거나 같은 가장 큰 정수를 반환한다.

---

## CEIL(), CEILING()

`CEIL()` 또는 `CEILING()`은 값을 올림한다.

```sql
SELECT CEILING(123.1);
```

결과:

```text
124
```

음수에서는 다음과 같이 동작한다.

```sql
SELECT CEILING(-123.9);
```

결과:

```text
-123
```

---

## TRUNCATE()

`TRUNCATE()`는 반올림 없이 숫자를 잘라낸다.

MySQL 기준 예시:

```sql
SELECT TRUNCATE(123.4567, 2);
```

결과:

```text
123.45
```

```sql
SELECT TRUNCATE(123.4567, 0);
```

결과:

```text
123
```

`ROUND()`는 반올림하지만, `TRUNCATE()`는 그냥 자른다는 점이 다르다.

---

## ABS()

`ABS()`는 절대값을 반환한다.

```sql
SELECT ABS(-100);
```

결과:

```text
100
```

---

## MOD()

`MOD()`는 나머지를 구할 때 사용한다.

```sql
SELECT MOD(10, 3);
```

결과:

```text
1
```

DB에 따라 `%` 연산자도 사용할 수 있다.

```sql
SELECT 10 % 3;
```

---

## POWER()

`POWER()`는 제곱을 구한다.

```sql
SELECT POWER(2, 3);
```

결과:

```text
8
```

---

## SQRT()

`SQRT()`는 제곱근을 구한다.

```sql
SELECT SQRT(16);
```

결과:

```text
4
```

---

## 숫자 처리 함수 실무 예시

평균 점수를 소수 둘째 자리까지 반올림할 때:

```sql
SELECT ROUND(AVG(score), 2)
FROM student;
```

가격을 100원 단위로 버릴 때:

```sql
SELECT FLOOR(price / 100) * 100
FROM product;
```

예를 들어 `12780`은 `12700`이 된다.

가격을 100원 단위로 올릴 때:

```sql
SELECT CEILING(price / 100) * 100
FROM product;
```

예를 들어 `12780`은 `12800`이 된다.

백분율을 계산할 때:

```sql
SELECT ROUND(success_cnt * 100.0 / total_cnt, 1) AS rate
FROM statistics;
```

# 2. 비트마스크 문제: DEVELOPERS와 SKILLCODES

## 문제 상황

`DEVELOPERS` 테이블에서 개발자의 등급을 구해야 한다.

등급 기준은 다음과 같다.

```text
A : Front End 스킬과 Python 스킬을 함께 가지고 있는 개발자
B : C# 스킬을 가진 개발자
C : 그 외의 Front End 개발자
```

조회해야 하는 컬럼은 다음과 같다.

```text
GRADE, ID, EMAIL
```

정렬 조건은 다음과 같다.

```text
GRADE 오름차순, ID 오름차순
```

---

## 핵심 개념: 비트마스크

이 문제에서 `SKILL_CODE`는 여러 스킬을 하나의 숫자로 표현한 값이다.

예를 들어 스킬 코드가 다음과 같다고 해보자.

| 스킬     | CODE | 2진수  |
| ------ | ---: | ---- |
| Python |    1 | 0001 |
| C#     |    2 | 0010 |
| React  |    4 | 0100 |
| Java   |    8 | 1000 |

어떤 개발자의 `SKILL_CODE`가 `5`라면 다음과 같다.

```text
5 = 0101
```

이는 다음 스킬을 가지고 있다는 의미다.

```text
Python = 0001
React  = 0100
```

즉,

```text
0101 = Python + React
```

---

## 비트 AND 연산

특정 스킬을 가지고 있는지 확인하려면 `&` 연산을 사용한다.

```sql
D.SKILL_CODE & S.CODE
```

예를 들어 개발자의 `SKILL_CODE`가 `5`이고 Python 코드가 `1`이라면:

```text
5      = 0101
Python = 0001
--------------
결과   = 0001
```

결과가 0보다 크므로 Python을 가지고 있다.

```sql
(D.SKILL_CODE & PY.CODE) > 0
```

반대로 C# 코드가 `2`라면:

```text
5  = 0101
2  = 0010
---------
0  = 0000
```

결과가 0이므로 C#은 가지고 있지 않다.

# 3. WITH를 사용한 풀이

## 전체 SQL

```sql
WITH FE AS (
    SELECT SUM(CODE) AS CODE
    FROM SKILLCODES
    WHERE CATEGORY = 'Front End'
),
PY AS (
    SELECT CODE
    FROM SKILLCODES
    WHERE NAME = 'Python'
),
CS AS (
    SELECT CODE
    FROM SKILLCODES
    WHERE NAME = 'C#'
)

SELECT
    CASE
        WHEN (D.SKILL_CODE & FE.CODE) > 0
         AND (D.SKILL_CODE & PY.CODE) > 0 THEN 'A'
        WHEN (D.SKILL_CODE & CS.CODE) > 0 THEN 'B'
        WHEN (D.SKILL_CODE & FE.CODE) > 0 THEN 'C'
    END AS GRADE,
    D.ID,
    D.EMAIL
FROM DEVELOPERS D
CROSS JOIN FE
CROSS JOIN PY
CROSS JOIN CS
WHERE
      ((D.SKILL_CODE & FE.CODE) > 0 AND (D.SKILL_CODE & PY.CODE) > 0)
   OR ((D.SKILL_CODE & CS.CODE) > 0)
   OR ((D.SKILL_CODE & FE.CODE) > 0)
ORDER BY GRADE, ID;
```

---

## WITH FE 이해하기

```sql
WITH FE AS (
    SELECT SUM(CODE) AS CODE
    FROM SKILLCODES
    WHERE CATEGORY = 'Front End'
)
```

`FE.CODE`는 여러 행이 아니라 `SUM(CODE)`의 결과이기 때문에 최종적으로는 1개의 값이다.

예를 들어 Front End 스킬 코드가 다음과 같다고 해보자.

| NAME    | CATEGORY  | CODE |
| ------- | --------- | ---: |
| React   | Front End |    1 |
| Vue     | Front End |    2 |
| Angular | Front End |    4 |

그러면:

```sql
SELECT SUM(CODE)
FROM SKILLCODES
WHERE CATEGORY = 'Front End';
```

결과는 다음과 같다.

```text
1 + 2 + 4 = 7
```

즉, `FE`는 다음과 같은 1행짜리 테이블이 된다.

| CODE |
| ---: |
|    7 |

따라서 아래 구문은:

```sql
D.SKILL_CODE & FE.CODE
```

실제로는 다음과 비슷한 의미가 된다.

```sql
D.SKILL_CODE & 7
```

---

## FE.CODE는 반복문처럼 동작하는가?

아니다.

`FE.CODE`는 Front End 스킬을 하나씩 반복해서 검사하는 것이 아니다.

이미 `SUM(CODE)`를 통해 Front End 스킬들의 코드를 하나의 비트 집합으로 합친 상태다.

예를 들어:

```text
React   = 0001
Vue     = 0010
Angular = 0100
----------------
합계     = 0111
```

이렇게 만들어진 값이 `FE.CODE`다.

그래서:

```sql
(D.SKILL_CODE & FE.CODE) > 0
```

은 다음 의미다.

```text
개발자가 Front End 스킬 중 하나라도 가지고 있는가?
```

---

## CASE 구문 해석

```sql
CASE
    WHEN (D.SKILL_CODE & FE.CODE) > 0
     AND (D.SKILL_CODE & PY.CODE) > 0 THEN 'A'
    WHEN (D.SKILL_CODE & CS.CODE) > 0 THEN 'B'
    WHEN (D.SKILL_CODE & FE.CODE) > 0 THEN 'C'
END AS GRADE
```

등급 판정 순서는 중요하다.

먼저 A를 검사한다.

```sql
WHEN Front End 보유 AND Python 보유 THEN 'A'
```

그다음 B를 검사한다.

```sql
WHEN C# 보유 THEN 'B'
```

마지막으로 C를 검사한다.

```sql
WHEN Front End 보유 THEN 'C'
```

`CASE`는 위에서 아래로 조건을 검사하고, 처음으로 참이 되는 조건의 결과를 반환한다.

따라서 A 조건을 먼저 검사해야 한다.

---

## WHERE 조건이 필요한 이유

```sql
WHERE
      ((D.SKILL_CODE & FE.CODE) > 0 AND (D.SKILL_CODE & PY.CODE) > 0)
   OR ((D.SKILL_CODE & CS.CODE) > 0)
   OR ((D.SKILL_CODE & FE.CODE) > 0)
```

이 조건은 A, B, C 중 하나라도 해당하는 개발자만 조회하기 위해 사용한다.

만약 이 조건이 없으면 `CASE` 결과가 `NULL`인 개발자도 조회될 수 있다.

문제에서는 "GRADE가 존재하는 개발자"만 조회하라고 했기 때문에 `WHERE`에서 걸러준다.

# 4. GROUP BY를 사용한 풀이

`GROUP BY`를 반드시 사용해야 하는 조건이 있다면, 개발자가 가진 스킬을 `SKILLCODES`와 조인해서 행으로 펼친 뒤 개발자별로 묶어서 등급을 계산할 수 있다.

---

## 전체 SQL

```sql
SELECT
    CASE
        WHEN SUM(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) > 0
         AND SUM(CASE WHEN S.NAME = 'Python' THEN 1 ELSE 0 END) > 0
            THEN 'A'
        WHEN SUM(CASE WHEN S.NAME = 'C#' THEN 1 ELSE 0 END) > 0
            THEN 'B'
        WHEN SUM(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) > 0
            THEN 'C'
    END AS GRADE,
    D.ID,
    D.EMAIL
FROM DEVELOPERS D
JOIN SKILLCODES S
    ON (D.SKILL_CODE & S.CODE) != 0
GROUP BY D.ID, D.EMAIL
HAVING
       (SUM(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) > 0
    AND SUM(CASE WHEN S.NAME = 'Python' THEN 1 ELSE 0 END) > 0)
    OR SUM(CASE WHEN S.NAME = 'C#' THEN 1 ELSE 0 END) > 0
    OR SUM(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) > 0
ORDER BY GRADE, D.ID;
```

---

## JOIN 구문 이해하기

```sql
JOIN SKILLCODES S
    ON (D.SKILL_CODE & S.CODE) != 0
```

이 조인은 개발자가 가지고 있는 스킬만 `SKILLCODES`와 연결한다.

예를 들어 개발자 ID가 1이고, 가진 스킬이 Python과 React라면 조인 결과는 다음처럼 펼쳐진다.

| ID | EMAIL                           | SKILL  | CATEGORY  |
| -: | ------------------------------- | ------ | --------- |
|  1 | [a@test.com](mailto:a@test.com) | Python | Back End  |
|  1 | [a@test.com](mailto:a@test.com) | React  | Front End |

즉, 한 명의 개발자가 여러 스킬을 가지고 있으면 여러 행으로 나온다.

---

## GROUP BY 이해하기

```sql
GROUP BY D.ID, D.EMAIL
```

조인 결과를 개발자별로 다시 묶는다.

예를 들어 조인 결과가 다음과 같다고 해보자.

| ID | SKILL  | CATEGORY  |
| -: | ------ | --------- |
|  1 | Python | Back End  |
|  1 | React  | Front End |
|  1 | Java   | Back End  |
|  2 | C#     | Back End  |
|  3 | Vue    | Front End |

`GROUP BY D.ID, D.EMAIL`을 하면 개발자별 그룹이 만들어진다.

```text
ID = 1 그룹: Python, React, Java
ID = 2 그룹: C#
ID = 3 그룹: Vue
```

---

## SUM(CASE WHEN ...) 이해하기

다음 구문은 조건부 집계라고 볼 수 있다.

```sql
SUM(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END)
```

의미는 다음과 같다.

```text
해당 개발자가 가진 스킬 중 Front End 스킬이 몇 개인가?
```

예를 들어 ID가 1인 개발자의 그룹이 다음과 같다고 하자.

| ID | SKILL  | CATEGORY  |
| -: | ------ | --------- |
|  1 | Python | Back End  |
|  1 | React  | Front End |
|  1 | Vue    | Front End |

각 행에서 `CASE` 결과는 다음과 같다.

| SKILL  | CATEGORY  | CASE 결과 |
| ------ | --------- | ------: |
| Python | Back End  |       0 |
| React  | Front End |       1 |
| Vue    | Front End |       1 |

그 다음 `SUM`을 하면:

```text
0 + 1 + 1 = 2
```

즉, Front End 스킬이 2개 있다는 뜻이다.

그래서 다음 조건은:

```sql
SUM(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) > 0
```

다음 의미가 된다.

```text
Front End 스킬을 하나 이상 가지고 있는가?
```

---

## SUM 내부의 CASE는 반복문처럼 동작하는가?

개념적으로는 반복문처럼 이해해도 된다.

정확히 말하면 SQL이 `GROUP BY`로 묶인 그룹 안의 각 행에 대해 `CASE`를 평가하고, 그 결과들을 `SUM`으로 합친다.

예를 들어 그룹 안에 세 행이 있다면:

```text
Python     → Front End 아님 → 0
React      → Front End 맞음 → 1
Vue        → Front End 맞음 → 1
```

그 후:

```text
SUM(0, 1, 1) = 2
```

가 된다.

따라서 이 패턴은 다음처럼 이해하면 된다.

```sql
SUM(CASE WHEN 조건 THEN 1 ELSE 0 END)
```

```text
그룹 안에서 조건을 만족하는 행의 개수
```

---

## MAX(CASE WHEN ...)로도 가능하다

이 문제에서는 개수가 중요한 것이 아니라 존재 여부만 중요하다.

따라서 `SUM()` 대신 `MAX()`를 사용할 수도 있다.

```sql
MAX(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END)
```

이 값은 다음 둘 중 하나다.

```text
1: Front End 스킬이 하나 이상 있음
0: Front End 스킬이 없음
```

그래서 아래처럼 쓸 수도 있다.

```sql
SELECT
    CASE
        WHEN MAX(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) = 1
         AND MAX(CASE WHEN S.NAME = 'Python' THEN 1 ELSE 0 END) = 1
            THEN 'A'
        WHEN MAX(CASE WHEN S.NAME = 'C#' THEN 1 ELSE 0 END) = 1
            THEN 'B'
        WHEN MAX(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) = 1
            THEN 'C'
    END AS GRADE,
    D.ID,
    D.EMAIL
FROM DEVELOPERS D
JOIN SKILLCODES S
    ON (D.SKILL_CODE & S.CODE) != 0
GROUP BY D.ID, D.EMAIL
HAVING
       (MAX(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) = 1
    AND MAX(CASE WHEN S.NAME = 'Python' THEN 1 ELSE 0 END) = 1)
    OR MAX(CASE WHEN S.NAME = 'C#' THEN 1 ELSE 0 END) = 1
    OR MAX(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END) = 1
ORDER BY GRADE, D.ID;
```

# 5. WITH 방식과 GROUP BY 방식 비교

## WITH 방식

```text
Front End 스킬 코드들을 먼저 하나의 값으로 합친다.
그 후 DEVELOPERS 테이블과 비교한다.
```

장점:

```text
비트마스크의 특성을 잘 활용한다.
쿼리가 비교적 간단하다.
Front End 전체를 한 번에 검사할 수 있다.
```

핵심:

```sql
SELECT SUM(CODE)
FROM SKILLCODES
WHERE CATEGORY = 'Front End'
```

이 결과가 하나의 비트 집합이 된다.

---

## GROUP BY 방식

```text
개발자가 가진 스킬을 행으로 펼친다.
개발자별로 다시 묶는다.
조건부 집계로 각 스킬 보유 여부를 계산한다.
```

장점:

```text
GROUP BY와 조건부 집계를 연습하기 좋다.
각 개발자가 어떤 종류의 스킬을 가지고 있는지 집계하기 쉽다.
```

핵심:

```sql
SUM(CASE WHEN 조건 THEN 1 ELSE 0 END)
```

이 구문은 그룹 안에서 조건을 만족하는 행의 개수를 센다.

# 7. 내가 헷갈렸던 부분 정리

## FE.CODE는 여러 개일까?

처음에는 `FE.CODE`가 Front End 스킬 여러 개를 반복해서 검사하는 것처럼 보였다.

하지만 실제로는 아니다.

```sql
SELECT SUM(CODE) AS CODE
FROM SKILLCODES
WHERE CATEGORY = 'Front End'
```

이 쿼리 결과가 1행 1값으로 나오기 때문에 `FE.CODE`는 하나의 숫자다.

즉:

```sql
D.SKILL_CODE & FE.CODE
```

는 반복문이 아니라, 이미 합쳐진 비트 집합과 한 번에 AND 연산하는 것이다.

---

## SUM(CASE WHEN ...)은 반복문처럼 생각해도 될까?

개념적으로는 반복문처럼 생각해도 된다.

```sql
SUM(CASE WHEN S.CATEGORY = 'Front End' THEN 1 ELSE 0 END)
```

이 구문은 그룹 안의 각 행에 대해:

```text
Front End면 1
아니면 0
```

을 만들고, 그 값들을 모두 합친다.

따라서 결과는:

```text
그 개발자가 가진 Front End 스킬 개수
```

가 된다.

# 8. 최종 기억 문장

```text
WITH 방식의 FE.CODE는 여러 Front End 스킬을 SUM으로 합친 하나의 비트 집합이다.
따라서 반복문처럼 도는 것이 아니라 한 번의 비트 AND 연산으로 검사한다.
```

```text
GROUP BY 방식의 SUM(CASE WHEN ...)은 그룹 안의 행들을 하나씩 평가한 뒤,
조건을 만족하면 1, 아니면 0으로 바꾸고 합산하는 조건부 집계다.
```

```text
비트마스크 문제에서는 & 연산 결과가 0보다 크면 해당 스킬을 가지고 있다고 판단한다.
```
