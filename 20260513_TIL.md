# MSSQL Trigger 확인 방법

## 1. 특정 테이블의 Trigger 조회

특정 테이블에 연결된 Trigger를 확인할 때 사용한다.

```sql
SELECT
    t.name AS trigger_name,
    OBJECT_NAME(t.parent_id) AS table_name,
    t.create_date,
    t.modify_date,
    t.is_disabled
FROM sys.triggers t
WHERE OBJECT_NAME(t.parent_id) = '테이블명';
```

예시:

```sql
SELECT
    t.name AS trigger_name,
    OBJECT_NAME(t.parent_id) AS table_name
FROM sys.triggers t
WHERE OBJECT_NAME(t.parent_id) = 'Orders';
```

---

## 2. 현재 DB의 전체 Trigger 조회

DB에 존재하는 모든 Trigger를 확인할 수 있다.

```sql
SELECT
    name,
    parent_class_desc,
    create_date,
    is_disabled
FROM sys.triggers;
```

---

## 3. Trigger 내부 SQL 확인

Trigger 안에 어떤 로직이 작성되어 있는지 확인할 때 사용한다.

```sql
EXEC sp_helptext '트리거명';
```

예시:

```sql
EXEC sp_helptext 'trg_Orders_Insert';
```

---

## 4. Trigger 이벤트 종류 확인

해당 Trigger가 INSERT / UPDATE / DELETE 중 어떤 이벤트에서 동작하는지 조회한다.

```sql
SELECT
    t.name AS trigger_name,
    te.type_desc
FROM sys.triggers t
JOIN sys.trigger_events te
    ON t.object_id = te.object_id;
```

---

## 5. Trigger 활성화 여부 확인

```sql
SELECT
    name,
    is_disabled
FROM sys.triggers;
```

| 값  | 의미     |
| --- | -------- |
| 0   | 활성화   |
| 1   | 비활성화 |

---

## 6. 가장 간단한 방법

테이블 기준으로 Trigger를 빠르게 조회할 수 있다.

```sql
sp_helptrigger '테이블명'
```

예시:

```sql
sp_helptrigger 'Orders'
```

---

# 정리

실무에서는 보통 아래 순서로 많이 확인하게 된다.

1. `sp_helptrigger` 로 Trigger 존재 여부 확인
2. `sp_helptext` 로 내부 로직 확인
3. `sys.trigger_events` 로 이벤트 종류 확인

간단한 조회는 `sp_helptrigger` 가 가장 편했고,
상세 분석은 `sys.triggers` 와 `sp_helptext` 조합이 유용했다.
