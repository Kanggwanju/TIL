# Java 생성자, 오버라이딩, 필드 초기화 정리

## 1. 생성자에서 `this()`와 `super()`의 관계

### 예제

```java
class Parent {
    Parent() {
        System.out.print("Parent-Const ");
    }
}

class Child extends Parent {
    Child() {
        System.out.print("Child-NoArg ");
    }

    Child(String name) {
        this();
        System.out.print("Child-" + name + " ");
    }
}
```

---

### 핵심 규칙

생성자의 첫 문장은 반드시 아래 둘 중 하나여야 한다.

```java
this(...)
```

또는

```java
super(...)
```

둘을 동시에 사용할 수는 없다.

---

### `super()` 자동 삽입 규칙

컴파일러는 생성자에

- `this(...)` 없음
- `super(...)` 없음

인 경우에만

```java
super();
```

를 자동 삽입한다.

예:

```java
Child() {
    System.out.print("Child-NoArg ");
}
```

↓

컴파일러가 내부적으로

```java
Child() {
    super();
    System.out.print("Child-NoArg ");
}
```

로 변환한다.

---

### `this()`가 있는 경우

```java
Child(String name) {
    this();
    System.out.print("Child-" + name + " ");
}
```

이미 첫 줄에 `this()`가 있으므로 컴파일러는 `super()`를 추가하지 않는다.

실행 순서:

```text
Child(String)
  ↓
this()
  ↓
Child()
  ↓
super()
  ↓
Parent()
```

출력:

```text
Parent-Const Child-NoArg Child-Test
```

## 2. 오버라이딩과 공변 반환 타입(Covariant Return Type)

### 가능

부모 메서드의 반환 타입보다 더 구체적인 타입을 반환할 수 있다.

```java
class Super {
    Object getData() {
        return new Object();
    }
}

class Sub extends Super {
    @Override
    Integer getData() {
        return 100;
    }
}
```

`Integer`는 `Object`의 하위 타입이므로 허용된다.

---

### 이유

```java
Super s = new Sub();
Object obj = s.getData();
```

컴파일러는 `Object`를 기대한다.

실제로는 `Integer`가 반환된다.

```java
Integer instanceof Object
```

이므로 안전하다.

---

### 용어

이것을

> 공변 반환 타입 (Covariant Return Type)

이라고 한다.

---

## 3. 오버라이딩 시 매개변수 타입 변경

### 불가능

```java
class Super {
    void print(Object obj) {}
}

class Sub extends Super {
    @Override
    void print(String obj) {}
}
```

컴파일 오류 발생.

---

### 이유

이것은 오버라이딩이 아니라 오버로딩이다.

부모의 메서드 시그니처:

```java
print(Object)
```

자식의 메서드 시그니처:

```java
print(String)
```

서로 다른 메서드로 취급된다.

---

### 오버라이딩 규칙

#### 반환 타입

하위 타입 허용

```java
Object
↓
Integer
```

가능

---

#### 매개변수 타입

반드시 동일

```java
Object
↓
String
```

불가능

---

## 4. 필드(Field)는 오버라이딩되지 않는다

### 메서드

메서드는 오버라이딩된다.

```java
Super s = new Sub();
s.method();
```

실행 시점의 실제 객체 타입(`Sub`) 기준으로 결정된다.

---

### 필드

필드는 오버라이딩이 아니라 숨김(Hiding)이다.

```java
Super s = new Sub();
System.out.println(s.value);
```

컴파일 시점의 참조 타입(`Super`) 기준으로 결정된다.

---

## 5. 생성 과정에서의 필드 초기화 순서

### 예제

```java
class Container {
    int value = 5;

    Container() {
        adjust();
    }

    void adjust() {
        value += 10;
    }
}

class Box extends Container {
    int value = 20;

    @Override
    void adjust() {
        value += 100;
    }
}
```

---

## `new Box()` 실행 순서

### 1단계: 메모리 할당

모든 필드 기본값

```text
Container.value = 0
Box.value = 0
```

---

### 2단계: 부모 필드 초기화

```java
int value = 5;
```

실행

```text
Container.value = 5
Box.value = 0
```

---

### 3단계: 부모 생성자 실행

```java
Container() {
    adjust();
}
```

---

### 4단계: 동적 바인딩 발생

실제로 호출되는 메서드

```java
Box.adjust()
```

---

### 5단계: Box.adjust() 실행

```java
value += 100;
```

여기서의 `value`

```java
Box.value
```

현재 값:

```text
Box.value = 0
```

실행 후:

```text
Box.value = 100
```

---

### 6단계: 부모 생성자 종료

현재 상태

```text
Container.value = 5
Box.value = 100
```

---

### 7단계: 자식 필드 초기화

```java
int value = 20;
```

실행

```text
Container.value = 5
Box.value = 20
```

기존의 100은 덮어써진다.

---

### 8단계: 자식 생성자 본문 실행

명시적 생성자가 없으므로 종료

---

## 최종 상태

```text
Container.value = 5
Box.value = 20
```

---

## 출력 결과

```java
Container c = new Box();
Box d = new Box();

System.out.println(c.value);
System.out.println(d.value);
```

### c.value

참조 타입:

```java
Container
```

따라서

```java
Container.value
```

출력

```text
5
```

---

### d.value

참조 타입:

```java
Box
```

따라서

```java
Box.value
```

출력

```text
20
```

---

### 최종 출력

```text
5
20
```

## 6. 생성자에서 오버라이딩 가능한 메서드를 호출하면 위험한 이유

예:

```java
class Parent {
    Parent() {
        init();
    }

    void init() {}
}
```

```java
class Child extends Parent {
    int value = 100;

    @Override
    void init() {
        System.out.println(value);
    }
}
```

실행 시

```java
new Child();
```

출력:

```text
0
```

---

### 이유

부모 생성자가 실행될 때

```java
Child.value
```

는 아직

```java
100
```

으로 초기화되지 않았다.

기본값

```java
0
```

상태이다.

---

### 권장 사항

생성자 내부에서 오버라이딩 가능한 메서드 호출은 피하는 것이 좋다.

---

# 핵심 한 줄 요약

- `this()`가 있으면 `super()`는 자동 삽입되지 않는다.
- 오버라이딩 시 반환 타입은 하위 타입으로 변경 가능(공변 반환 타입).
- 오버라이딩 시 매개변수 타입은 변경 불가.
- 메서드는 오버라이딩되지만 필드는 숨김(Hiding)이다.
- 생성 중의 메서드 호출은 동적 바인딩이 적용된다.
- 생성자에서 오버라이딩 가능한 메서드 호출은 초기화 순서 문제 때문에 위험하다.
