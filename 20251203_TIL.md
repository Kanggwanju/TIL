## 1️⃣ 트리 순회의 개념

### 💡 정의

트리 순회(Traversal)는 **트리 구조에 포함된 모든 노드를 한 번씩 방문하는 체계적인 방법**을 말한다.
트리는 **계층적 구조**이므로, 배열이나 리스트처럼 단순 반복으로는 전체 탐색이 불가능하다.
따라서 **재귀(Recursion)** 을 이용해 하위 트리(서브 트리)까지 순차적으로 접근한다.

## 2️⃣ 이진 트리(Binary Tree)의 구조

```plaintext
        (10)
       /    \
     (20)   (30)
     / \       \
   (40)(50)   (60)
```

* **루트(Root)**: 10
* **리프(Leaf)**: 40, 50, 60
* **트리의 높이**: 3
* **노드 수**: 6

---

## 3️⃣ 순회 방법의 종류

### 1️⃣ 전위 순회 (Preorder Traversal)

> 방문 순서: **중 → 왼 → 오**

```c
void preorder(TreeNode* node) {
    if (node == NULL) return;
    printf("[%d] ", node->data);
    preorder(node->left);
    preorder(node->right);
}
```

**활용 예시:**

* 트리 복사 (부모를 먼저 처리해야 구조를 유지할 수 있음)

**출력 결과:**
`[10] [20] [40] [50] [30] [60]`

---

### 2️⃣ 중위 순회 (Inorder Traversal)

> 방문 순서: **왼 → 중 → 오**

```c
void inorder(TreeNode* node) {
    if (node == NULL) return;
    inorder(node->left);
    printf("[%d] ", node->data);
    inorder(node->right);
}
```

**활용 예시:**

* **이진 탐색 트리(Binary Search Tree)** 의 데이터 출력 시 **오름차순 정렬 결과**를 얻을 수 있음

**출력 결과:**
`[40] [20] [50] [10] [30] [60]`

---

### 3️⃣ 후위 순회 (Postorder Traversal)

> 방문 순서: **왼 → 오 → 중**

```c
void postorder(TreeNode* node) {
    if (node == NULL) return;
    postorder(node->left);
    postorder(node->right);
    printf("[%d] ", node->data);
}
```

**활용 예시:**

* **트리 전체 삭제** (자식부터 해제하고 부모를 마지막에 삭제해야 함)

**출력 결과:**
`[40] [50] [20] [60] [30] [10]`

## 4️⃣ 전체 코드 요약

```c
TreeNode* root = createNode(10);
TreeNode* n20 = createNode(20);
TreeNode* n30 = createNode(30);
TreeNode* n40 = createNode(40);
TreeNode* n50 = createNode(50);
TreeNode* n60 = createNode(60);

linkNodes(root, n20, n30);
linkNodes(n20, n40, n50);
linkNodes(n30, NULL, n60);

printf("전위 순회 결과:\n");
preorder(root);

printf("\n중위 순회 결과:\n");
inorder(root);

printf("\n후위 순회 결과:\n");
postorder(root);
```

## 🎯 최종 정리

| 순회 방식             | 처리 순서     | 대표적 활용           | 출력 예시             |
| ----------------- | --------- | ---------------- | ----------------- |
| **전위(Preorder)**  | 중 → 왼 → 오 | 트리 복사            | 10 20 40 50 30 60 |
| **중위(Inorder)**   | 왼 → 중 → 오 | 정렬된 데이터 출력 (BST) | 40 20 50 10 30 60 |
| **후위(Postorder)** | 왼 → 오 → 중 | 트리 삭제, 후처리       | 40 50 20 60 30 10 |

## 💬 오늘의 소감

오늘은 **이진 트리 순회(Traversal)** 를 직접 구현하며,
**재귀 호출이 트리 구조를 탐색하는 데 얼마나 자연스럽게 작동하는지** 깨달았다.
전위, 중위, 후위 순회의 순서만 바꿔도 완전히 다른 결과가 나오는 점이 흥미로웠고,
특히 중위 순회에서 이진 탐색 트리의 데이터가 자동으로 오름차순으로 출력되는 것이 인상적이었다.
