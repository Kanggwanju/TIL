## 1️⃣ 탐색 알고리즘의 개념

### 💡 정의

* **탐색(Search)**: 데이터 집합 내에서 원하는 값을 찾는 과정
* 대표적인 탐색 방식

  1. **선형 탐색 (Linear Search)** → 순차적으로 확인
  2. **이진 탐색 (Binary Search)** → 정렬된 데이터를 절반씩 줄이며 탐색

---

## 2️⃣ 선형 탐색 (Linear Search)

### 💻 코드 예시

```c
int linearSearch(int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        printf("반복문 실행 %d회차\n", i + 1);
        if (arr[i] == target) {
            return i;
        }
    }
    return -1;
}
```



### 🔍 개념 정리

* 배열의 **처음부터 끝까지** 순차적으로 비교
* 찾으면 인덱스 반환, 못 찾으면 `-1` 반환
* **정렬되어 있지 않아도 사용 가능**

### ⏱️ 시간 복잡도

| 구분 | 복잡도  | 설명             |
| -- | ---- | -------------- |
| 최선 | O(1) | 첫 번째 원소에서 찾을 때 |
| 평균 | O(N) | 중간 정도에서 찾을 때   |
| 최악 | O(N) | 마지막 또는 없음      |

### 장단점

| 장점        | 단점            |
| --------- | ------------- |
| 구현이 매우 단순 | 데이터가 많으면 비효율적 |
| 정렬 필요 없음  | 탐색 횟수가 많음     |

## 3️⃣ 이진 탐색 (Binary Search)

### 💻 코드 예시

```c
int binarySearch(int arr[], int n, int target) {
    int low = 0, high = n - 1, mid;
    int i = 0;

    while (low <= high) {
        printf("반복문 실행 %d회차\n", i + 1);
        mid = (low + high) / 2;

        if (arr[mid] == target)
            return mid;
        else if (arr[mid] < target)
            low = mid + 1;
        else
            high = mid - 1;

        i++;
    }

    return -1;
}
```



### 🔍 개념 정리

* **정렬된 배열**을 전제로 함
* 중앙값과 비교 후, **절반씩 탐색 범위를 줄여감**
* 매 반복마다 탐색 범위가 1/2로 줄어듦

### ⏱️ 시간 복잡도

| 구분 | 복잡도      | 설명             |
| -- | -------- | -------------- |
| 최선 | O(1)     | 첫 번째 비교에서 찾을 때 |
| 평균 | O(log N) | 매번 반씩 줄어듦      |
| 최악 | O(log N) | 끝까지 줄여도 없음     |

### 장단점

| 장점             | 단점               |
| -------------- | ---------------- |
| 탐색 속도가 매우 빠름   | 데이터가 정렬되어 있어야 함  |
| 데이터 크기가 커도 효율적 | 삽입·삭제 시 정렬 유지 필요 |

## 4️⃣ 예제 비교

| 항목         | 선형 탐색          | 이진 탐색             |
| ---------- | -------------- | ----------------- |
| **데이터 조건** | 정렬 불필요         | 반드시 정렬 필요         |
| **탐색 방식**  | 처음부터 끝까지 순차 탐색 | 중앙값 기준 반으로 나누며 탐색 |
| **시간 복잡도** | O(N)           | O(log N)          |
| **속도**     | 느림             | 빠름                |
| **구현 난이도** | 쉬움             | 중간                |
| **데이터 규모** | 소규모에 적합        | 대규모에 적합           |

## 5️⃣ 시각적 이해

**선형 탐색 흐름**

```
[18, 7, 25, 30, 11, 9]
 ↑   ↑   ↑   ↑   ↑   ↑
(순서대로 확인)
```

**이진 탐색 흐름**

```
[1, 3, 5, 7, 9, 11, 13, 15]
 중간 → 7 → 오른쪽 → 11 (탐색 완료)
```

| **이진 탐색** | 빠름, 효율적       | 정렬 필요    | O(log N) |
