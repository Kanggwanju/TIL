# 🗓️ 2026년 6월 9일 TIL - SQL 재귀 CTE(Recursive CTE)

## 배운 계기

프로그래머스의 ECOLI_DATA 문제를 풀면서 "3세대 찾기", "세대별 개체 수 구하기" 문제를 접했다.

처음에는 Self Join으로 해결하려고 했지만, 세대가 몇 단계까지 존재하는지 알 수 없는 경우에는 JOIN만으로 해결할 수 없다는 것을 알게 되었다.

이때 사용하는 것이 **재귀 CTE(Recursive CTE)** 이다.

---

# 1. 재귀 CTE란?

재귀 CTE는 자기 자신을 참조하면서 반복적으로 데이터를 조회하는 SQL 문법이다.

트리(Tree) 구조나 계층(Hierarchy) 구조를 탐색할 때 사용한다.

예시:

* 조직도
* 카테고리 구조
* 댓글 대댓글 구조
* ECOLI_DATA의 세대 구조

# 2. ECOLI_DATA 예시

## 테이블

| ID | PARENT_ID |
| -- | --------- |
| 1  | NULL      |
| 2  | 1         |
| 3  | 1         |
| 4  | 2         |
| 5  | 2         |
| 6  | 4         |

### 세대 구조

```text
1
├── 2
│   ├── 4
│   │   └── 6
│   └── 5
└── 3
```

세대로 표현하면

```text
1세대 : 1
2세대 : 2, 3
3세대 : 4, 5
4세대 : 6
```

# 3. 재귀 CTE 기본 구조

```sql
WITH RECURSIVE CTE_NAME AS (

    -- 시작점(Anchor)

    SELECT ...

    UNION ALL

    -- 반복부(Recursive)

    SELECT ...
    FROM ...
    JOIN CTE_NAME ...

)

SELECT *
FROM CTE_NAME;
```

재귀 CTE는 크게 두 부분으로 구성된다.

1. Anchor
2. Recursive

---

# 4. 세대 구하기

```sql
WITH RECURSIVE GEN AS (

    SELECT
        ID,
        PARENT_ID,
        1 AS GENERATION
    FROM ECOLI_DATA
    WHERE PARENT_ID IS NULL

    UNION ALL

    SELECT
        E.ID,
        E.PARENT_ID,
        G.GENERATION + 1
    FROM ECOLI_DATA E
    JOIN GEN G
        ON E.PARENT_ID = G.ID

)

SELECT *
FROM GEN;
```

결과

| ID | PARENT_ID | GENERATION |
| -- | --------- | ---------- |
| 1  | NULL      | 1          |
| 2  | 1         | 2          |
| 3  | 1         | 2          |
| 4  | 2         | 3          |
| 5  | 2         | 3          |
| 6  | 4         | 4          |

# 6. 재귀 CTE를 코드로 이해하기

재귀 CTE는 사실상 아래 의사코드와 같다.

```python
gen = [1세대]

while True:
    new_rows = 자식찾기(gen)

    if not new_rows:
        break

    gen += new_rows
```

즉,

```text
부모
↓
자식
↓
손자
↓
증손자
↓
...
```

를 반복적으로 탐색하는 구조이다.

# 7. 자식이 없는 개체 찾기

조건

```text
자신의 ID가 다른 행의 PARENT_ID로 등장하지 않음
```

쿼리

```sql
SELECT ID
FROM ECOLI_DATA
WHERE ID NOT IN (
    SELECT PARENT_ID
    FROM ECOLI_DATA
    WHERE PARENT_ID IS NOT NULL
);
```

예시 결과

```text
3
5
6
```

# 8. 세대별 자식 없는 개체 수 구하기

## 풀이 순서

### 1단계

재귀 CTE로

```text
ID
GENERATION
```

정보 생성

### 2단계

자식 없는 개체 추출

### 3단계

세대별 GROUP BY

---

## 최종 쿼리

```sql
WITH RECURSIVE GEN AS (

    SELECT
        ID,
        PARENT_ID,
        1 AS GENERATION
    FROM ECOLI_DATA
    WHERE PARENT_ID IS NULL

    UNION ALL

    SELECT
        E.ID,
        E.PARENT_ID,
        G.GENERATION + 1
    FROM ECOLI_DATA E
    JOIN GEN G
        ON E.PARENT_ID = G.ID

)

SELECT
    COUNT(*) AS COUNT,
    GENERATION
FROM GEN
WHERE ID NOT IN (
    SELECT PARENT_ID
    FROM ECOLI_DATA
    WHERE PARENT_ID IS NOT NULL
)
GROUP BY GENERATION
ORDER BY GENERATION;
```

# 핵심 정리

## Self Join

* 세대 수를 알고 있을 때 사용
* 3세대 찾기 가능

```sql
E1 -> E2 -> E3
```

---

## Recursive CTE

* 세대 수를 모를 때 사용
* N세대까지 탐색 가능

```sql
WITH RECURSIVE
```

---

## 자주 나오는 유형

* 조직도
* 카테고리 트리
* 댓글/대댓글
* ECOLI_DATA 세대 문제
* 그래프 탐색

## 기억할 공식

### 1세대 찾기

```sql
PARENT_ID IS NULL
```

### 자식 찾기

```sql
E.PARENT_ID = G.ID
```

### 자식 없는 개체

```sql
ID NOT IN (
    SELECT PARENT_ID
    FROM ...
)
```

### 세대 증가

```sql
G.GENERATION + 1
```
